# Contribuer à Archimède

## Comment contribuer ?

1. **Lire la RFC** (`docs/rfc/v0.2.md`) et l'architecture de référence (`docs/architecture/reference-v0.1.md`) pour comprendre la vision et les choix structurants.
2. **Ouvrir une issue** pour discuter d'une contribution potentielle (nouveau module, nouvelle intégration, correction de bug, amélioration de documentation).
3. **Forker le dépôt**, créer une branche, et soumettre une Pull Request (PR).

## Règles de contribution

- **Documentation** : toute nouvelle fonctionnalité ou modification significative doit être documentée.
- **Tests** : les contributions doivent inclure des tests (unitaires, d'intégration, ou de reproductibilité).
- **Reproductibilité** : les exemples et démonstrations doivent être exécutables en une commande, avec des données publiques.
- **Revue de code** : toute PR majeure est relue par au moins deux contributeurs.

## Structure du code (à venir)

Le noyau (`archimede-core`) sera stable. Les domaines (`archimede-energy`, `archimede-materials`, etc.) seront des plugins indépendants.
